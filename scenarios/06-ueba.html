👤 What is UEBA?

User and Entity Behavior Analytics builds a behavioral baseline for each user — typical login times, accessed systems, source locations. Any deviation is flagged.

Most effective for detecting Insider Threats (valid credentials, abnormal behavior) and Compromised Accounts (credentials stolen, used by attacker).

🗺 Impossible Travel

A user authenticates from Frankfurt at 10:00 AM, then from Singapore at 10:10 AM. Physically impossible → alert. Could mean: stolen credentials, VPN use (verify separately), shared account (policy violation).

⚠
Key Signals: After-hours login · Weekend access · New sensitive system access · Login from new country/IP range · Sudden data volume spike
SPL · After-Hours Login Detection Windows Security · EventID 4624
index=* sourcetype=WinEventLog:Security EventCode=4624 earliest=-24h | eval hour = strftime(_time,"%H") | eval dow = strftime(_time,"%A") | eval after_hours = if(hour<"08" OR hour>"19" OR dow="Saturday" OR dow="Sunday", "YES", "NO") | where after_hours="YES" | stats count as logins, values(hour) as hours, values(ComputerName) as systems by Account_Name | sort -logins
ComponentExplanation
strftime(_time, "%H")Extract event hour (00-23). Represents event time — NOT index time
dow = Saturday/SundayWeekend logins from 9-to-5 accounts are high-value behavioral anomalies
values(hour)Shows exact hours logged in — useful for pattern analysis in investigation
🔍 Phase 1 — User Profile Analysis ▾
Review user's normal login pattern for last 30 days
Check HR records — on notice, on leave, recently resigned?
Check VPN logs — was the source IP a VPN exit node?
Review post-login activity — what files and systems were accessed?
🌳 Insider Threat vs. Compromised Account
Did you contact the user? Did they confirm this login?
✓ Yes, confirmed → Possible policy violation — report to HR
✗ No / denies → Compromised Account — disable immediately + IR
🚨 Response ▾
If compromised: disable account and force password reset on all accounts
If insider: notify HR, preserve evidence (do NOT warn the user)
Check all activity from the account for the past 90 days
