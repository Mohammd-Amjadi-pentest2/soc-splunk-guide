🔎 What is IOC Hunting?

After receiving new Threat Intelligence (ISAC, VirusTotal, CISA advisory), the analyst proactively searches historical logs to check if the IOCs appear in their environment.

Always search at least 90 days back — the attacker may have been present for weeks before detection.

🌱 Pyramid of Pain

Hash (easiest to change) → IP → Domain → Network/Host Artifacts → Tools → TTPs (hardest to change)

Focus hunting on TTPs and tools — they're more durable. Hashes change in minutes.

💡
IOC types: IP Address · Domain/FQDN · File Hash (SHA256) · URL Pattern · User-Agent String · Registry Key
SPL · Malicious IP Hunt (Multi-Source, 90 Days) All Sources · earliest=-90d
index=* earliest=-90d (src_ip="185.220.101.45" OR dest_ip="185.220.101.45" OR src_ip="194.165.16.33" OR dest_ip="194.165.16.33") | stats count as hits, min(_time) as first_seen, max(_time) as last_seen, values(src_ip) as internal_hosts, values(sourcetype) as log_sources by dest_ip | eval first_seen = strftime(first_seen,"%Y-%m-%d %H:%M") | sort -hits
SPL · File Hash Hunt (Sysmon EventID 1+7) 90-Day Lookback
index=* sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational (EventCode=1 OR EventCode=7) earliest=-90d | eval sha256 = lower(replace(Hashes,".*SHA256=([^,]+).*","\1")) | where sha256 IN ("a3f5e2b8c1d4e9f7a2b5c8d1e4f7a0b3", "b7d2e5f8a1c4b7e0f3a6b9c2d5e8f1a4") | stats count, values(Computer) as hosts, values(Image) as paths by sha256
📥 Phase 1 — IOC Preparation ▾
Validate IOC source — CISA, VirusTotal with multiple detections, trusted ISAC
Categorize by type: IP, Domain, Hash, URL
Set lookback window — minimum 90 days
🌳 IOC Hit Response
Did the IOC hunt return any hits?
✗ No → Document search scope and time window. Add IOCs to blocklists.
✓ Yes → Continue
How many internal hosts communicated with the malicious IOC?
✓ 1 host → Investigate that host. Likely contained.
✗ >1 host → Assume breach. Activate full Incident Response Plan.
🚨 Active Hunt Response ▾
On any hit: immediately expand investigation to all activity from that host
Check surrounding time window ±2 hours for correlated events
Identify all other internal hosts in the same subnet
