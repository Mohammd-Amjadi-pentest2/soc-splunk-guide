🔒 What is Persistence?

After gaining access, attackers establish persistence to ensure access survives system reboots and user logoffs. Without persistence, the malware dies on reboot. Attackers often set multiple persistence methods as backup channels.

🔑 Common Persistence Methods

Registry Run Keys: HKCU/HKLM\Software\Microsoft\Windows\CurrentVersion\Run — executes on every login
Scheduled Tasks (EventID 4698): Time- or event-triggered execution
New Services (EventID 7045): Runs as SYSTEM — highest privilege
WMI Subscriptions: Persistent event-triggered execution in memory

🔴
Red Flag: Legitimate software installs to Program Files. A Run Key pointing to %TEMP% or AppData\Roaming = almost certainly malicious.
SPL · Registry Run Key Modification (Sysmon EventID 13) Registry persistence detection
index=* sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational EventCode=13 earliest=-24h | where match(TargetObject, "(?i)(CurrentVersion\\\\Run|Winlogon\\\\Userinit)") | eval suspicious = if( match(Details,"(?i)(temp|appdata|powershell|cmd\.exe|wscript)"), "YES","NO") | where suspicious="YES" | table Computer, User, TargetObject, Details, Image
SPL · New Scheduled Task (EventID 4698) Windows Security · -24h
index=* sourcetype=WinEventLog:Security EventCode=4698 earliest=-24h | eval suspicious = if( match(TaskContent,"(?i)(powershell|cmd|wscript|temp|appdata|http)"), "SUSPICIOUS","NORMAL") | where suspicious="SUSPICIOUS" | table ComputerName, Account_Name, TaskName, TaskContent
🔍 Phase 1 — Identify & Analyze ▾
Identify the executable path set in the persistence mechanism
Extract file hash and check in VirusTotal immediately
Determine when persistence was established — correlate with other events
Check if same pattern exists on other hosts
🌳 Persistence Threat Assessment
Is the file hash detected as malicious in VirusTotal?
✓ Yes → TRUE POSITIVE — Quarantine + Isolate + Full forensics
✗ No → Continue
Is the executable in a Temp/AppData directory (not Program Files)?
✓ Yes → HIGH SUSPICION — Investigate parent process that created it
✗ No → Check digital signature and file origin carefully
🚨 Response Actions ▾
Remove malicious Registry Key or disable Scheduled Task
Quarantine malicious file — do NOT delete, preserve for forensics
Check currently running processes — is the malware currently active?
Take memory dump before any system changes
