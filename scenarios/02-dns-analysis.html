DNS Tunneling & DGA Detection
DNS Logs · DNSSense/DNSEye · NXDOMAIN Analysis
DNS LogsDNSSense/DNSEyeDNS TunnelingDGAT1071.004

🌊 What is DNS Tunneling?

Attackers encode data inside DNS queries to create a covert channel. Since DNS (port 53) is almost never blocked by firewalls, it provides an ideal exfiltration path. Tools: iodine, dnscat2.

Key indicator: unusually long subdomains (>50 chars), high query rate to one domain, rare record types (TXT, NULL, ANY).

📈 What is DGA?

Malware generates pseudo-random domain names to contact its C2 server. The operator pre-registers upcoming names. If one is blocked, the next one is tried.

Key indicator: high NXDOMAIN rate from one host (>30/hr), random-looking domains (e.g., xk9p2mqa.ru), short TTL. Families: Conficker, Emotet, Dridex.

⚠
Log Artifacts: Long subdomain (>50 chars) · High NXDOMAIN rate · Regular interval queries (beaconing) · TXT/NULL record types
SPL · DNS Tunneling Detection DNS Logs · earliest=-1h
index=* sourcetype="dns" earliest=-1h | eval subdomain = replace(query,"\.[^.]+\.[^.]+$","") | eval sub_len = len(subdomain) | where sub_len > 50 | stats count as tunnel_queries, avg(sub_len) as avg_len by src_ip, query_type | where tunnel_queries > 20 | eval sev = case(avg_len>100,"CRITICAL",avg_len>70,"HIGH",1=1,"MEDIUM") | sort -tunnel_queries
ComponentExplanation
eval sub_lenCalculate subdomain length — DNS tunneling inflates this with encoded data
where sub_len > 50Subdomains over 50 chars are extremely rare in legitimate DNS traffic
case() sevAuto-classify: >100 avg chars = almost certainly active tunneling
SPL · DGA Detection (NXDOMAIN Rate) DNS Logs · -1h
index=* sourcetype="dns" reply_code="NXDOMAIN" earliest=-1h | stats count as nx_count, dc(query) as uniq_domains by src_ip | where nx_count > 30 | eval dga_score = round(uniq_domains/nx_count*100,2) | eval verdict = if(dga_score>80,"HIGH - DGA","MEDIUM") | sort -nx_count
🔍 Phase 1 — Validation ▾
Check subdomain length — over 50 chars is suspicious, over 100 is critical
Check NXDOMAIN rate — over 30/hr from one host = DGA suspect
Look for regular query intervals (beaconing pattern)
🌳 DNS Decision Tree
Is the domain flagged malicious in VirusTotal / Threat Intel?
✓ Yes → True Positive — Block domain at DNS + Escalate
✗ No → Continue
Is subdomain length >50 with Base64/hex pattern?
✓ Yes → High confidence Tunneling — Block + Host forensics
✗ No → Analyze NXDOMAIN rate and query frequency
🚨 Phase 2 — Response (TP Confirmed) ▾
Block malicious domain in DNS Sinkhole or Firewall
Isolate originating host from network
Take memory dump for forensic analysis
Identify all other internal hosts querying the same domain
