🐠 PowerShell as an Attack Vector

PowerShell is pre-installed, trusted by default, and extremely powerful. Attackers abuse it for:

Download cradles — fetch and execute remote code in memory (IEX + DownloadString)
Encoded commands — -EncodedCommand base64 to bypass logging
AMSI bypass — disable antivirus scanning of scripts
Fileless execution — run entirely in memory, no disk artifact

🏫 LOLBins — Living Off the Land

Legitimate Windows tools abused for malicious purposes:
certutil.exe — download files from internet
mshta.exe — execute HTA/VBScript from URL
regsvr32.exe — execute DLL or remote scripts
wscript.exe — run JavaScript or VBScript
These bypass application whitelisting because they are signed Microsoft binaries.

🚨
Parent Process is Critical: PowerShell from WINWORD.EXE = Macro attack. From svchost.exe = Very suspicious WMI execution.
SPL · Malicious PowerShell Risk Scoring Sysmon EventID 1 · -1h
index=* sourcetype=XmlWinEventLog:Microsoft-Windows-Sysmon/Operational EventCode=1 (Image="*\\powershell.exe" OR Image="*\\pwsh.exe") earliest=-1h | eval enc = if(match(CommandLine,"(?i)(-enc|-EncodedCommand)"),30,0) | eval dl = if(match(CommandLine,"(?i)(DownloadString|WebClient|iex)"),40,0) | eval bypass = if(match(CommandLine,"(?i)(-nop|-bypass|-WindowStyle Hidden)"),20,0) | eval amsi = if(match(CommandLine,"(?i)(amsi|AmsiScanBuffer)"),50,0) | eval risk_score = enc+dl+bypass+amsi | where risk_score > 20 | eval verdict = case(risk_score>=70,"CRITICAL",risk_score>=40,"HIGH",1=1,"MEDIUM") | sort -risk_score
SignalScoreWhy
-EncodedCommand+30Obfuscating the command to evade script block logging
DownloadString / IEX+40Download and execute remote code in memory — no disk artifact
-nop / -bypass+20Bypassing execution policy and hiding window
amsi / AmsiScanBuffer+50Attempting to disable AV scanning — highest confidence signal
🔍 Phase 1 — Command Analysis ▾
Read full CommandLine — if encoded, Base64-decode it immediately
Check ParentImage — WINWORD/EXCEL as parent = Macro Attack
Did the process make network connections? (Sysmon EventID 3)
Did it create files on disk? (Sysmon EventID 11)
🌳 PowerShell Threat Assessment
Is ParentImage a user application (Word, Excel, Outlook)?
✓ Yes → HIGH CONFIDENCE Macro Attack — Isolate immediately
✗ No → Continue
Does CommandLine contain download cradle (IEX + WebClient)?
✓ Yes → CRITICAL: Remote code execution — Block C2 + Isolate
✗ No → Check risk_score and investigate further
🚨 Immediate Response (TP Confirmed) ▾
Isolate host — disconnect from network
Kill the suspicious process (do NOT reboot — preserve memory)
Take memory dump for forensic analysis
Block C2 destination IP/domain in Firewall
